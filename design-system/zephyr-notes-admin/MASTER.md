# Zephyr Notes Admin Design System

> 页面专项规则位于 `design-system/zephyr-notes-admin/pages/`。页面规则可以细化本文件，
> 但不能破坏这里定义的信息架构、可访问性和视觉层级。

## 产品定位

Zephyr Notes Admin 是内容运营工作台，不是营销网站。界面优先支持快速扫描、连续处理和
可追溯操作，避免装饰性大标题、嵌套卡片和与任务无关的说明文案。

## 信息架构

| 分组 | 模块       | 主要职责                                       |
| ---- | ---------- | ---------------------------------------------- |
| 总览 | 运营工作台 | 聚合内容进度、互动治理、Agent 待确认和最近动态 |
| 创作 | 文章       | 管理文章、分类、标签与发布进度                 |
| 创作 | Agent 助手 | 通过对话和可确认任务辅助内容运营               |
| 互动 | 评论       | 结合文章、作者和回复上下文治理评论             |
| 互动 | 留言板     | 管理留言可见性、置顶和软删除                   |
| 系统 | 站点配置   | 维护首页、导航、公告和 Agent 治理策略          |
| 系统 | 审计日志   | 追踪所有后台写操作和治理记录                   |

- 根路由必须进入运营工作台，不能默认进入某个单一工具。
- 导航名称使用业务语言，不暴露运行时、工作流或基础设施术语。
- 页面标题说明当前职责；操作说明放在控件附近，不使用大段教学文案。

## 视觉原则

- **风格：** 克制的玻璃拟态。透明表面、柔和模糊和轻阴影用于表达层级。
- **品牌：** 使用 `apps/admin/src/styles.css` 中的 AZ Notes 青绿色主色，暖玫瑰仅作辅助。
- **字体：** 使用项目内置寒蝉全圆体和系统回退，不加载外部展示字体。
- **圆角：** 控件和面板最大 8px；不使用胶囊形容器承载普通文本。
- **密度：** 工作台密度为 8/10。同一屏优先展示更多可操作信息。
- **图标：** 统一使用 Lucide。熟悉的图标操作必须提供 `aria-label` 和 `title`。

## 色彩与表面

代码中的 CSS 变量是唯一颜色来源：

- `--background` / `--foreground`：页面底色和主要文本。
- `--primary`：主操作、焦点和品牌强调。
- `--muted` / `--muted-foreground`：次级信息和非活跃状态。
- `--glass-surface`：普通工作面板。
- `--glass-surface-strong`：侧栏、顶部栏和需要更高不透明度的表面。
- `--glass-border`：必要的弱边界。
- `--shadow-glass` / `--shadow-glass-strong`：两级玻璃阴影。

### 暗色模式

- 主要通过色调透明度和阴影区分表面，不依赖连续描边。
- 大面板、侧栏和顶部栏不显示实体边框。
- 表格、输入框等操作区域可保留 4-12% 中性边界。
- 不使用彩色轮廓或硬质选中竖线；键盘焦点环必须保持清晰。

### 亮色模式

- 使用偏冷的浅灰绿背景和白色半透明表面。
- 阴影表达层级，边框只用于输入、表格和状态边界。
- 主文本与背景至少达到 WCAG AA 对比度。

## 布局

- 桌面端使用 `280px + minmax(0, 1fr)` 的固定侧栏布局。
- 主内容最大宽度 1600px，页面间距按 `12 / 20 / 28px` 响应式增长。
- 顶部栏保持紧凑，只承载页面标题、职责说明和必要全局动作。
- 移动端侧栏转为顶部水平导航；列表和表单按单列排列。
- 固定格式区域必须使用明确的 grid track、最小宽度和换行策略。

## 组件规范

### 工作面板

- 使用 `ManagementSurface` 作为治理、设置和分类页面的顶层容器。
- `ManagementHeader` 承载标题、短说明、数量和主要刷新/保存操作。
- `ManagementToolbar` 承载搜索、状态筛选和提交动作。
- `ManagementList` 使用弱分隔行，不把每条记录做成独立描边卡片。
- 禁止在工作面板内再嵌套装饰性卡片。

### 指标卡

- 只用于可比较的重复指标。
- 固定图标区域、标签和 tabular 数字，避免加载时发生布局位移。
- 每组最多四项；更多信息进入列表或详情面板。

### 表单

- 字段始终有可访问标签；占位符不能替代标签。
- 保存动作位于当前配置面板顶部或编辑区底部，不重复出现。
- 危险操作使用 destructive 变体并要求确认。
- 二元状态使用 Checkbox；小型选项集使用 Select 或分段控件。

### 按钮

- 主按钮用于当前视图最重要的动作，每个局部区域通常不超过一个。
- 常见工具动作优先图标按钮；不熟悉或不可逆命令使用图标加文字。
- 点击目标至少 40px，触屏关键操作至少 44px。

## 状态与动效

- 加载、空状态、错误和成功反馈必须占用稳定空间，不能导致主布局跳动。
- 状态变化使用 150-300ms CSS transition。
- Hover 只调整颜色、阴影或极小位移，不改变元素尺寸。
- 遵守 `prefers-reduced-motion` 和 `prefers-reduced-transparency`。
- 所有交互控件必须有可见的 `focus-visible` 状态。

## 页面规则

- `pages/dashboard.md`：运营总览的数据、优先级和快捷入口。
- `pages/articles.md`：文章列表、筛选、状态和编辑入口。

- [ ] 所有指标和列表来自真实后台 API，不使用 mock 回退。
- [ ] 暗色模式没有重复的连续描边。
- [ ] 亮色和暗色文本对比度清晰。
- [ ] 375px、768px、1024px、1440px 下无横向溢出。
- [ ] 键盘焦点、语义标签和图标按钮名称完整。
- [ ] 加载、空、错误和危险确认状态完整。
